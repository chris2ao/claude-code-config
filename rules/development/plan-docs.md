# Plan Documentation

## Plan Output Location

When creating implementation plans, always save them as markdown files:

1. Save to `docs/plans/` in the current project's working directory
2. If the `docs/plans/` directory does not exist, create it
3. Use descriptive filenames: `<topic>-plan.md` (e.g., `auth-refactor-plan.md`)
4. Plans should be committed alongside the code they describe

This applies to all projects, regardless of framework or language.

**Note:** This overrides the superpowers writing-plans default of `docs/superpowers/plans/`. The superpowers skill respects user preferences for plan location.
